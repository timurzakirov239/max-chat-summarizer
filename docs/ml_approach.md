# ML-подход: Суммаризация и ранжирование

## 1. Двухэтапная суммаризация

### Этап 1: Extractive (TextRank)
1. Сообщения объединяются в единый документ
2. TF-IDF векторизация предложений
3. Косинусное сходство → граф предложений
4. PageRank ранжирование
5. Отбор top-K предложений (~60% сжатие)

### Этап 2: Abstractive (ruT5-base)
1. Отобранные предложения подаются в fine-tuned `ai-forever/ruT5-base`
2. Генерация: beam search (4 beams), no_repeat_ngram_size=3
3. Результат: связный пересказ на русском языке

### Почему 2 этапа?
- ruT5-base ограничен 512 токенами — extractive сжимает вход
- Extractive убирает шум, abstractive создаёт связный текст
- Дедупликация + abstractive исключают повторы

## 2. Обучение модели

### Датасеты
1. **Gazeta** (HuggingFace: `IlyaGusev/gazeta`) — 60K+ пар (статья → краткое содержание)
2. **Собственный датасет чатов** — domain adaptation (~500-1000 примеров)

### Метрики
- ROUGE-1, ROUGE-2, ROUGE-L
- BERTScore

## 3. Ранжирование важности

### Признаки
| Признак | Описание |
|---------|----------|
| reply_count | Количество ответов |
| message_length | Нормализованная длина |
| has_question | Наличие вопроса |
| has_link | Наличие ссылки |
| word_count | Количество слов (log) |
| unique_word_ratio | Доля уникальных слов |
| position_score | Позиция в потоке |
| has_attachments | Наличие вложений |

### Модель
- XGBoost classifier на размеченных данных
- Эвристический fallback (взвешенная сумма признаков)
